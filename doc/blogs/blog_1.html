Live nodeList

For some reason(fuck DOM standard),when we use some method for query nodeList or HTMLCollection(It’s another fucking thing),we will get a live nodeList or HTMLCollection. The live is mean this collection or array like object (fuck the javascript, why so many “surprise” ?) maybe change when you update the node property.>

The problem will boom when we iterate this nodeList, if we change the node property in iterate. I show the code follow for explain it.

let nodeList_0=document.getElementsByClassName("certain_class")
R.map(
  (x)=>x.className="another_class" ,
  nodeList_0
)//you will get type error for get className for undefined

What happen? <br> <br>
Becuse the nodeList_0 is a live nodeList, so when you change the node’s property className, the nodeList_0 will update,assume the nodeList_0 's length is 3, now the length will update to 2. Becasue the getElemnetsByClassName only got two element. So when you access nodeList[2], you will got a undefined.
<br>
So, how to avoid this problem? <br>

  function(){
      this.className = "another_class"
  }
  )

  R.identity,
